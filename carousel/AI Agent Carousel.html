Start here
Your first agent should be boring.
Most people start way too big — read the whole inbox, update the CRM, book the meetings. That's exactly the wrong place to start. Boring is what you can see, debug, and fix when it breaks.
03NEXT →
The real lesson
You're not learning leads. You're learning a shape.
We'll build one tiny thing: a lead-enrichment workflow. Spreadsheet in → research each business → structured results → spreadsheet out.
That same shape runs support automation, document processing, research assistants, and a hundred things you'll build next.
04NEXT →
The mental model
It's an assembly line — not a genie.
Most of your "AI agent" isn't AI. Loading the file, running search, writing results back — all ordinary code.
The model only does the judgment: decide what to search, read the messy results, return clean fields.
05NEXT →
The architecture
One row. One sequence. Repeat.
01Read a row from the spreadsheet.
02Ask the model what to search for.
03Run the search; hand results back.
04Get a structured record. Validate it.
05Write it back. Move to the next row.
A straight line you could draw on a napkin.
06NEXT →
Give it a shape
Don't let the model freestyle the answer.
If software has to use the answer, the answer needs structure. Hand it a schema and make it fill every field — same shape, every time.
{ "business_summary": "string", "likely_contact": "string | null", "confidence": "high | med | low", "needs_human_review": true }
07NEXT →
Scope discipline
Run it on five rows.
Not five thousand.
Five rows surface almost every problem you'll hit at scale — generic names, wrong-company matches, missing sites, and a model that sounds far more confident than the evidence warrants.
Five rows is your control room.
08NEXT →
The curriculum
The failures aren't bugs in your learning. They are the learning.
Generic name? Fold city + state into the query.
Wrong company? Make it match name, location, and site first.
Invents an owner? Return null unless public sources prove it.
Enforce the rules that matter in code — not in the prompt.
09NEXT →
The takeaway
Your first agent shouldn't be impressive. It should be understandable.
Narrow job. One tool. Tight scope. Structured output. Honest uncertainty. Improve it after you watch it fail.
That's how you stop playing with AI and start managing it.

# Markdown Sample Guide

This is a sample Markdown file. It shows the most common formatting options you’ll use when writing docs, briefs, prompts, campaign notes, or AI instructions.

---

## 1. Headings

Headings create structure in a document.

# Heading 1: Main title
## Heading 2: Major section
### Heading 3: Subsection
#### Heading 4: Smaller subsection
##### Heading 5: Rarely used
###### Heading 6: Very rarely used

---

## 2. Paragraphs

Markdown paragraphs are just regular text.

To create a new paragraph, leave a blank line between blocks of text.

This is a second paragraph. It is separate from the first because there is a blank line above it.

---

## 3. Bold, Italic, and Emphasis

Use bold when something is important.

**This text is bold.**

Use italic for light emphasis.

*This text is italic.*

You can combine them.

***This text is bold and italic.***

---

## 4. Bullet Points

Use bullet points when order does not matter.

- Campaign brief
- Audience research
- Landing page copy
- Email sequence
- Social media posts

You can also nest bullet points.

- Product Launch
  - Homepage copy
  - Email announcement
  - Social posts
- Sales Enablement
  - One-pager
  - Demo script
  - Objection handling notes

---

## 5. Numbered Lists

Use numbered lists when order matters.

1. Review the campaign brief
2. Draft the landing page copy
3. Send to product for review
4. Make revisions
5. Publish the page

Nested numbered lists also work.

1. Create the campaign brief
   1. Define the audience
   2. Write the core message
   3. List the proof points
2. Create the content assets
   1. Landing page
   2. Email sequence
   3. Social posts

---

## 6. Links

Links use this format:

[Link text](https://example.com)

Example:

[Visit our website](https://example.com)

You can also include links inside sentences, like this:

Read the [brand guidelines](https://example.com) before writing final copy.

---

## 7. Images

Images are similar to links, but they start with an exclamation point.

![Alt text describing the image](https://example.com/image.png)

Example:

![Product dashboard screenshot](https://example.com/dashboard.png)

The alt text should briefly describe the image.

---

## 8. Block Quotes

Use block quotes for quoted text, notes, or callouts.

> This is a block quote.
> It can be used for customer quotes, reminders, or important notes.

Example:

> “The new dashboard saves our team hours every week.”
> — Customer interview quote

---

## 9. Inline Code

Inline code is useful for filenames, commands, tools, labels, or short technical references.

Use the `CLAUDE.md` file to explain what is inside a project folder.

The file should be saved as `campaign-brief.md`.

---
